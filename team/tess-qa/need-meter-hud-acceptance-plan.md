## Silent-killer seams to guard (carry-forward + new)

1. **Warmth regression under generalization.** The #1 risk: refactoring `SurvivalHud` to N bars subtly changes warmth's FLOOR rule / segment count / plate alpha / band colors / right-to-left empty. **Guard:** the existing warmth tests stay green AND the shipped warmth bar is visually identical. This is the explicit regression-guard gate.
2. **One bar silently static** (the dual-spawn "pickup_count > 0 passed during the whole dual-spawn era" class). Two of three bars wire correctly; the third polls a null ref / subscribes to the wrong need / never repaints. **Guard:** drive EACH need independently and assert ONLY its bar moves; assert all three are bound to the CORRECT need (not all three reading warmth).
3. **Surface mismatch with the need tickets.** This HUD binds `Current01`/`IsCritical`/`Changed`. If hunger/thirst shipped a divergent surface (the hunger/thirst G1 shared-base risk), the HUD compiles against warmth but null-refs / mis-reads the others. **Guard:** the hard-dep surface check (AC1 of the preflight) + bind-to-correct-need test.
4. **Palette drift from Uma's pins.** The bars must be Uma's exact world-derived sub-1.0 colors with DISTINCT empties. **Guard:** EditMode palette-RGB assertions + shipped eye-dropper spot-check against `need-meter-ui-direction.md` §3/§4/§5.
5. **BUILD-stamp occlusion + belt collision** (Uma §1/§8 soak watch-items). The column must not cover the top-right BUILD stamp (load-bearing for every soak) and must not crowd the bottom-center belt at small window widths. **Guard:** shipped capture at the soak window size; confirm stamp legible + no overlap.
6. **Per-frame allocation in critical-breathe / satisfaction tweens** (Uma §9: alloc-free, one shared eased lerp / one-shot tweens). A `new Color`/string alloc per frame in `OnGUI` is a GC-churn regression. **Guard:** code-review the breathe/pulse path for per-frame allocation.
