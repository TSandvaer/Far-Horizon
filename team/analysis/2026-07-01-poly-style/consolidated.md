# Far Horizon — Consolidated Poly-World Improvement Plan (visual + performance)

**Date:** 2026-07-01. **Sources:** two sibling deep-analyses (visual-quality art angle; performance/tech angle) of the generated poly world at `c:\Trunk\PRIVATE\Far-Horizon` (branch `orch/coordination`, HEAD `844e8d5`; three look files — `GradientSkybox.shader`, `QualityPassGen.cs`, `WorldBootstrap.cs` — cited from `origin/main` where the sun-disk PR #194 lives). This document is self-contained: a reader does not need the source reports.

**Honest headline (perf):** the scene is *small* for a Windows desktop target — <1M verts total, ~zero texture memory (vertex-color world), 2 realtime lights. The real configured costs, in order: (1) the shadow pipeline (4096 map / 4 cascades / 220u distance / ~1,300 casters), (2) an unused full-screen Opaque Texture copy every frame, (3) ~1,350 unbatched unique-mesh draws (multiplied by cascades), (4) the 14.2MB embedded-mesh `Boot.unity`. CPU scripting and GC are basically healthy. Classic optimizations that DON'T matter here: LODs, occlusion culling, texture compression, Update()-count micro-tuning — don't spend on them.

**Honest headline (visual):** the world already matches the art bar on sky, sun, sea colour, rocks, bushes, pond, mountains-silhouettes, fog/post. The gaps vs the inspiration board: monochrome ground carpet, single-tone single-species forest, lumpy-potato cloud undersides, statically-lit (never glinting) sea, wide pale foam wash, empty beaches, smooth-ramp cliffs, near-invisible sparse grass, barren vista shelves, no fauna.

## 1. Locked constraints (apply to EVERY item below)

Sponsor-locked or bug-scarred — **not defects, do not "fix":**

1. **Seed-42 island** outline, scatter placements, waterline — byte-locked (tests assert it). New prop classes = NEW seed salts (`seed+777/888/999` precedent, `LowPolyZoneGen.cs:1012,1052,1092`); per-instance variation on existing props = position-derived streams (`leanRnd` precedent, `:1161-1162`). Never add/remove draws on an existing `System.Random` stream.
2. **Pond: everything** (colour, stillness, foam-off, recess, collar) — 9+ soak rounds (#130). If sea glints ship, the pond material keeps `_GlintStrength 0`.
3. **Sand warm anchors** + damp-only-on-cliffs (`LowPolyZoneGen.cs:47-49, 853-860`); **sea teal anchors** `WaterShallow/Deep` (pixel-sampled ×3); **sun 18° / size 0.95 / warm-white** (accepted 2026-06-30); **cloud 3-value cyans**; **fog colour == SkyHorizon** seam-kill (single-constant binding — never fork it).
4. **Smooth-normal welded terrain** (never flat-shade it); **grass/bushes stationary, sway on tree canopies only** (#172); `_FoamAmount` master-gate semantics (pond OFF depends on it); outward-winding + explicit per-face normals on all flat-shaded meshes.
5. **Gamma color space is look-locked** — technically wrong for the HDR+bloom+tonemap stack, but every palette constant was soaked against gamma output. Document as deliberate (one line in `unity6-mastery.md`); revisit only in an explicit look-overhaul milestone. Zero desktop perf impact either way.
6. **Committed-generated assets:** every generator/config change ships ONLY after `BootstrapProject.Run` re-bake + committing `Boot.unity`/`Assets/Settings/*` — a code-only PR ships nothing. URP/renderer assets are regenerated by `ConfigureUrp` every bootstrap (`BootstrapProject.cs:163-247`) — fixes go in code, never by hand-editing the .asset (silently reverted).
7. **Single CI Unity build slot** — each regen PR occupies it and re-commits ~14MB of scene; batch related regens into few PR waves (§5).
8. **Colour discipline:** sub-1.0 all channels; only *intentional* sparkle (SEA-1 spec) may cross the bloom threshold 1.02. Near-neutral tints through `QuantizeFine`.
9. **Process:** Predict-Before-Soak + gameplay-cam capture evidence for anything visual; new dials go on the dev-console (`WorldLookTunables`); Danish-keyboard-safe keys only.

## 4. Tension resolutions (quality ↔ perf, stated explicitly)

- **T-A. Tonal variation vs batching.** All hue/tone variation (TRE-1 trees, BSH-1 bushes, GRD-1 ground, VIS-1 vista caps) is **vertex-color-baked, never per-material** — material count stays ~15-25 quantize-cached and the SRP Batcher keeps working. A per-material tint would multiply materials per instance; rejected.
- **T-B. Grass densification vs draw calls/shadow cost.** The grass meadow overhaul (GRS-1/2/3) lands **only after** the caster policy (R2a) and static-batching (R3-1) plumbing exist, so blades ship `ShadowCastingMode.Off` + `BatchingStatic` **from birth**; density is delivered by **patch masking** (concentrate tufts in GRD-1's lush patches; accept-probability ∝ patch value) so patches read dense while total count stays near the current budget. Profiler A/B on a `-development` build (S2a) before/after.
- **T-C. New geometry (pines, beach props, flowers) vs scene-file growth + single build slot.** `Boot.unity` grows with every unique mesh; the sea weld (G2) buys back ~5-6MB of headroom *first*; new-content regens are **batched into few PR waves** (§5) so the 14MB re-commit and the CI slot are paid once per wave, not per item. If world density ever scales 2-3×, switch new prop classes to small shared-mesh pools (R3-2 / G1-2) — soak-gated because per-instance mesh jitter is part of the approved look.
- **T-D. Cloud shadows (alive world) vs shadow pipeline being the #1 cost.** Never enable real shadow casting on clouds against the 4096/4-cascade stack (and at 18° sun elevation real shadows land far sideways, possibly off-island). If pursued (Tier 3), use 2-3 large soft drifting **fake decals** behind a console toggle, evaluated only after R2a/R2b settle the shadow budget.
- **T-E. Sea sparkle vs transparent overdraw + pond lock.** SEA-1's glints cost ~3-4 ALU + one `pow` in an already-running transparent frag — negligible; the transparent-queue decision itself stays as-is (deliberate, bounded, depth-fade foam requires it). Guardrails: `_GlintStrength` is a console dial defaulting modest, the **pond material pins 0**, and only the glint sparkle may cross bloom threshold 1.02.
- **T-F. Shadow trimming vs the "long warm shadows" art bar.** Split R2 in two: R2a removes ~530 *visually invisible* casters (grass/sticks/pebbles/berries — a 0.5u tuft shadows nothing at orbit framing) with zero look change, machine-gated by capture compare. R2b (distance 220→~130, cascades 4→3) **changes what the player can see** and re-opens the soaked boundary-flicker percepts — separate PR, Sponsor soak, only after S2a profiling confirms the shadow share.
- **T-G. Foam crisping vs Sponsor-praised "foam on all edges".** FOM-1 changes foam *distribution* (thin bright surf line + faint wash), not *presence* — total coverage stays similar; the coast is soak-scarred, so it ships alone with a Predict-Before-Soak line.
- **T-H. Doc-vs-config drift (Forward+, IL2CPP, gamma, static-batch rule).** Resolve by **documenting the deliberate deviation now, flipping at a named trigger later**: Forward+/GRD with the night/torches milestone (zero win at 2 lights today); IL2CPP when a second CI lane exists (it ~doubles build time on the scarcest resource); gamma at an explicit look-overhaul; the mastery doc's "don't static-batch" rule is conditioned on GRD being the batching path — record the R3-1 exception rationale when it lands. Never resolve drift by hand-editing generated assets (S4: `ConfigureUrp` reverts them).
