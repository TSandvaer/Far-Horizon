## Runtime shipped-build instruments (read live ground truth from the BUILT exe)

These are `Assets/Scripts/Runtime/*VerifyCapture.cs` components, inert unless launched with their `-verify*` flag, wired into `Boot.unity` by `MovementCameraScene`. They exist because the editor/headless layer LIES about live runtime behavior (editor-vs-runtime divergence; headless `Time.deltaTime≈0` stalls the Animator/agent — unity-conventions §Headless). Run via the matching `.github/workflows/scripts/verify_*_gate.sh` (windowed, NOT batchmode — ScreenCapture + a live Animator need a real swapchain).

| Instrument | Flag / gate script | Purpose |
|---|---|---|
| `SneakVerifyCapture.cs` | `-verifySneak` / `verify_sneak_gate.sh` (+ `-sneakNoFootSync` control) | 86caa3kur sneak-walk SMOOTHNESS + LOOP-HITCH. Holds W (walk baseline) → W+Ctrl (the crouch sneak); logs the per-frame root-step CoV (the movement-layer stutter metric Devon's smooth-direct-drive fixed) AND — the re-soak attempt-2 extension — DUMPS the live layer-0 Animator ground truth (`[SneakTrace]`: state hash + clip + normalizedTime + effective playback speed + in-transition + #186 LocoSpeedMul) and an `AnimLoopTrace` verdict that NAMES the loop-hitch candidate: #1 clip loop-seam / #2 foot-sync stall (CrouchWalk has NO speedParameter → ruled out, re-confirmed empirically) / #3 state re-entry (AnyState→CrouchWalk flap restarts the clip). `-sneakNoFootSync` forces `footSync` off at boot (CastawayCharacter.Awake) = the candidate-#2 disconfirming control. The animator-state accessors live on `CastawayCharacter` (`CurrentStateHash` / `CurrentStateNormalizedTime` / `CurrentStateEffectiveSpeed` / `CurrentClipName` / `IsInTransition`). Diagnostic, not pass/fail — the gate uploads the trace; a human reads the verdict, then fixes the named cause. |
