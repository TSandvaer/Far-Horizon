# tools/debug — instrument registry

One-line index of durable debug/build instruments. Reuse before rebuilding.

| Tool | Purpose |
|---|---|
| `blender_mcp_send.py` | Send a command to the running BlenderMCP (ahujasid) addon over its TCP socket (localhost:9876). Modes: `code <file>` (execute_code), `scene` (get_scene_info), `shot <out.png>` (viewport screenshot). The transport for all Far Horizon Blender-MCP modeling when the `mcp__blender__*` tools aren't wired into the agent session. |
| `bl_01_setup.py` | Weapon-pack scene setup: clean scene, metric units, 1.8m char reference, writes the locked 9-hex `weapon_palette.png`, builds the shared `WeaponPalette` material. Run once per modeling session. |
| `bl_02_axe.py` | Builds the hero-axe geometry (haft + faceted red head matching inspiration `21h08_08`): hawk-beak poll w/ top notch, fanned cutting edge, grip-through-head. Shade Smooth + all-edges Mark Sharp. |
| `bl_03_uv.py` | Assigns palette-block UVs by island (haft flood-fill) + geometric edge-strip detection. **Head color knob:** `scene['head_variant']` in {`red`,`steel`} switches the head main/shadow palette slots (Sponsor reconsidering red vs steel, 2026-06-19). |
| `bl_04_finalize_export.py` | Merge-by-distance, recalc normals, set grip-midpoint origin (z=0.45), apply rot/scale (NOT location), export `wpn_axe_01.fbx` with spec-exact settings (-Y fwd / Z up / Normals Only / Apply-Transform OFF / triangulated). |
| `bl_05_hero_render.py` | Eevee hero 3/4 render of the axe (blade-left / beak-right framing) -> `axe_hero_render.png`. |
| `bl_06_save.py` | Saves the working `.blend` to `Assets/Art/Props/WeaponPack/weapon_set_src.blend`. |
| `bl_07_fix_palette.py` | Rewrites `weapon_palette.png` with TRUE sRGB hex bytes (manual PNG encoder, no PIL) so Unity reads the exact locked hexes; reloads + re-packs the Blender image datablock. |
| `bl_10_knife_sword_spear.py` | Builds the matched `wpn_knife_01` / `wpn_sword_01` / `wpn_spear_01` to the look-locked axe style (faceted, Shade Smooth + Mark-Sharp-all, shared `WeaponPalette`, UVs parked on palette blocks). Honors the Sponsor locks: spear-tip STONE/flint, sword edge full-length one-side tapering before the crossguard. Grip-origin at (0,0,0), blade +Z. |
| `bl_11_export_set.py` | Merge-by-distance + recalc-normals + zero-location + export `wpn_knife_01/wpn_sword_01/wpn_spear_01.fbx` to `Assets/Art/Props/WeaponPack/` with spec-exact FBX settings (-Y fwd / Z up / FBX-unit-scale / Apply-Transform OFF / Normals Only). |
| `bl_12_weapon_quality_pass.py` | QUALITY PASS (supersedes bl_10) raising knife/sword/spear to the axe's bar after the #100 soak (too-small + under-detailed). Scales them up + adds modeled facets: BROAD faceted blades with a raised central spine ridge (lens x-section, broad face along X toward camera), full white edge-highlight rim plane, chunky beveled crossguard, segmented wrapped grip, faceted pommel; spear point = knapped STONE leaf (flake-scar facets like the axe head). Tris 84/108/100 (was 37/51/40). Holds the Sponsor locks (knife+sword STEEL, axe+spear STONE; sword edge tapers off before guard). |
| `bl_13_family_render.py` | Eevee family-lineup render (axe + knife + sword + spear, grips on a common base line, neutral slate bg + white key/fill suns) -> `weapon_family_render.png` for the Sponsor quality-pass confirm. |
| `bl_14_axe_head_shrink.py` | (HISTORY) directional head-shrink attempt that scaled a vert SUBSET about a pivot — caused the "chipping" the Sponsor rejected; superseded by uniform-scale approaches. Kept for the audit trail; do not re-run. |
| `bl_15_axe_head_clean.py` | (HISTORY) rebuilt a clean head island + uniform-scaled it; the head was later REVERTED to the restored 4208067 stone wedge (commit db8990b). Operated on the `.blend`; superseded by bl_17 (which bakes the restored FBX directly). |
| `bl_16_probe_axe_fbx.py` | DIAGNOSTIC (read-only). Imports `wpn_axe_01.fbx` + dumps ground-truth geometry: bounds, connected components (head=widest-X island, haft, lash band), and a Z-band X-span scan to LOCATE the real head<->haft junction. Run before any head-resize bake so the cut matches the real geometry, not an assumed z-fraction. |
| `bl_16b_probe_head_pivot.py` | DIAGNOSTIC (read-only). Locates the head component's base/seating + dumps head width by Z slice + the canonical head max-width metric, so the bake picks the correct uniform-scale pivot + has a before/after width number. |
| `bl_17_axe_head_bake_065.py` | BAKE the stone axe HEAD at a UNIFORM factor (default 0.65x, Sponsor's pick; `-- <factor>` overrides) DIRECTLY into `wpn_axe_01.fbx` — because the runtime head dial was broken across all inputs. Replicates `HeldWeaponCycleDebug.ResolveAxeHead/ApplyAxeHead` EXACTLY (long axis Z, junction = boundsMin + 0.50·span = the dial's `headJunctionFraction`; head = the 104 verts above the junction; uniform scale about the haft-centreline junction pivot). Imports the restored FBX fresh (idempotent, no compounding), preserves the haft/lash/grip-origin/+Z axis + the single `WeaponPalette` slot, re-exports with the -Y/Z/FBX-unit/Normals-Only contract. Prints head-width before/after + a haft-unchanged proof. Re-run after `git checkout` of the restored FBX to re-bake at a new factor. |
| `bl_18_axe_rebake_coaxial_1p5x.py` | RE-BAKE `wpn_axe_01.fbx` for the PR #100 re-soak: (1) RIGID-ROTATE the 110 head verts coaxial about the head-base mount centroid — kills the 20.14° dogleg to ~2.7° (an 87% reduction; iterates the blended centroid-line + binning-independent half-split axes to their stable fixed point), preserving head shape/size byte-identical-modulo-rotation (pairwise-distance invariant asserted, delta ~3e-6); (2) shorten the haft 2.0x→1.5x (scale the 18 haft verts' Z 0.75× about the fixed head-base, ratio→1.50). Preserves grip origin (0,0,0), single `WeaponPalette` slot, faceted normals (no recalc), §8 export. Emits FRONT+SIDE ortho renders (visual ground truth) to `team/devon-dev/rebake_diag_86cabh907/`. Idempotent vs the COMMITTED FBX; `git checkout` the FBX before re-running (else haft re-scales 0.75 again). Companion: `team/devon-dev/bl_axe_bend_diagnose_86cabh907.py` (the diagnose-only ortho+axis script that found the dogleg). **(SUPERSEDED by `bl_19` for the coaxial method — bl_18's centroid-line axis left a 2.71° residual the Sponsor still read as "bend"; bl_19 zeroes the haft-top→head-centroid MOUNT line instead → 0.0198°.)** |
| `bl_19_axe_coaxial_length_variants.py` | SHAFT-LENGTH PICKER bake (86cabh907; supersedes bl_18's coaxial method). Imports the committed `wpn_axe_01.fbx`, RIGID-ROTATES the 110 head verts FULLY COAXIAL by zeroing the **haft-top→head-centroid MOUNT line** to +Z about the **haft-top junction point** (NOT the head-base centroid-line bl_18 used — that left 2.71° because it measured the head's internal stack, not the visible mount tilt; the FRONT ortho showed the head mass swung off-haft). Junction angle → **0.0198°** (≤0.5°); rigid (pairwise-dist invariant, delta 3.55e-6 → head shape/size locked). Then produces **four length variants** of the original short haft (1.1x/1.2x/1.3x/1.4x, head locked, haft scaled about the fixed head-base) → `wpn_axe_01_len11..len14.fbx`. Prints the re-derived `HeroAxeCoaxialHeadHeightFromTipU=0.50247` (§9 rotation corollary). Emits per-variant FRONT ortho renders + the candidate-method comparison renders to `team/devon-dev/coaxial_diag_86cabh907/` (cand_1mountline.png is the chosen coaxial read). Idempotent vs the committed FBX; does NOT mutate the shipped `wpn_axe_01.fbx`. The runtime picker (`HeldAxeLengthPicker`, key `[L]`) swaps the held axe through these so the Sponsor picks the length in-hand. Companion diagnostics: `team/devon-dev/_diag_head_axis_methods.py`, `_diag_coaxial_candidates.py`, `_diag_render_candidates.py`. |

Render outputs (`axe_*.png`, `_*.png`, `_*.py`, `*.blend1`) are gitignored build/throwaway artifacts.
